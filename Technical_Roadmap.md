# 教育专家 - 技术执行路线图 (Technical Roadmap)

**致开发团队**：本项目核心是一个垂直于育儿场景的 AI 对话应用。我们的目标是“极简交互、深度专业”。以下是完成该项目的四个执行阶段。

---

## 阶段 1：基础设施与模型集成 (Infrastructure & LLM)
**目标**：搭建起能够跑通基础对话的后端架构。

-   **核心任务**：
    1.  **后端选型**：使用 Python (FastAPI/Django) 搭建 RESTful API。
    2.  **模型接入**：集成 OpenAI (GPT-4o) 或 DeepSeek 等高逻辑推理模型，配置 API 中转或直接调用。
    3.  **流式响应 (Streaming)**：必须实现 Stream 模式输出，确保用户在提问后能实时看到 AI 的回复，降低等待焦虑。
-   **程序员注意**：初期不需要复杂的数据库，先保证接口能接收字符串并返回 AI 生成的内容。

## 阶段 2：Prompt 工程与育儿逻辑注入 (Prompt Engineering)
**目标**：让 AI 听起来像真正的育儿专家，而不是通用的聊天机器人。

-   **核心任务**：
    1.  **System Prompt 定义**：编写高标准的系统指令。
        -   *角色设定*：儿童教育专家，懂伦理、懂心理、语调温柔且坚定。
        -   *逻辑链条*：先共情家长 -> 分析行为 -> 给出话术 -> 避坑建议。
    2.  **上下文管理**：设计有效的对话记忆机制。家长描述一个复杂事件可能分为多轮对话，AI 必须记住之前的背景。
    3.  **安全过滤 (Safety Layer)**：设置敏感词过滤，防止 AI 给出伤害儿童、体罚或违背法律的错误建议。

## 3. 阶段 3：极致简单的 UI 开发 (MVP Frontend)
**目标**：开发一个家长“秒上手”的移动端界面。

-   **核心任务**：
    1.  **框架选择**：推荐微信小程序或 Flutter。对于中国家长，小程序是门槛最低的入口。
    2.  **功能点实现**：
        -   一个置顶的“育儿求助”对话入口。
        -   支持文本输入（必选）和语音转文字输入（建议，方便忙碌家长）。
        -   历史记录存储：本地或云端保存过往的咨询，方便家长回看教育方案。
-   **UI 要求**：色彩暖色调，界面清爽，没有任何多余的广告或二级菜单。

## 4. 阶段 4：场景优化与压力测试 (Testing & Optimization)
**目标**：确保在极端案例（如孩子走丢、紧急伤害）下 AI 的建议依然稳健。

-   **核心任务**：
    1.  **场景模拟测试**：编写 20 个极端育儿案例，测试 AI 的回答质量（由产品经理与开发共同审核）。
    2.  **性能优化**：处理并发请求，确保响应速度。
    3.  **埋点分析**：统计家长最常问的问题类型（如：性格、安全、学习、习惯），为下一版本迭代提供数据支持。

---

### 技术栈建议 (Stack)
-   **Frontend**: Uni-app / Flutter / 微信小程序原生
-   **Backend**: Python 3.10+ / FastAPI
-   **Database**: PostgreSQL / Redis (存储对话上下文)
-   **LLM API**: GPT-4o-mini 或 DeepSeek-V3
